# 90-Day AI Transition Plan Template

Use this template to create a focused transition plan.

## Days 1-30: Orientation And Foundations

Goals:

1. Choose your target AI role.
2. Strengthen Python and data workflow basics.
3. Learn ML terminology and project workflow.
4. Choose one domain or workflow pain point worth building around.

Outputs:

- one public notes document
- one small Python or notebook project
- one role-specific learning plan
- one short market scan of target roles and recurring job requirements

## Days 31-60: Core AI Building Skills

Goals:

1. Learn supervised ML and evaluation basics.
2. Learn neural net and LLM fundamentals.
3. Build one meaningful applied project.
4. Define a simple evaluation rubric and failure log.

Outputs:

- one ML or analytics project
- one LLM or agent-style project
- written evaluation notes for both
- one short note explaining why the chosen use case deserves AI

## Days 61-90: Portfolio And Positioning

Goals:

1. Improve one project to production quality.
2. Publish clean READMEs and case studies.
3. Update resume, profile, and interview stories.
4. Turn one project into a visible market signal.

Outputs:

- 2-3 public portfolio artifacts
- updated GitHub profile and LinkedIn positioning
- one concise story for each project: problem, design, evaluation, lessons
- one public post, demo, or founder-style opportunity thesis

## Weekly Rule

Every week should include:

1. learning
2. building
3. documenting
4. reflecting on gaps
5. checking whether the work is becoming more credible, more trustworthy, and more useful

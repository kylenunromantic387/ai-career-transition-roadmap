# 180-Day AI Transition Plan Template

Use this if you want a deeper, more realistic transition plan.

## Phase 1: Orientation And Foundations (Days 1-45)

Goals:

1. choose your target role
2. build Python and data workflow confidence
3. understand AI terminology and core workflow
4. identify one domain wedge where your current background gives you leverage

Deliverables:

- a written role target
- one public study log
- one small coding or notebook exercise repo
- one written market scan covering recurring skills and role expectations

## Phase 2: ML And Deep Learning Foundations (Days 46-90)

Goals:

1. complete a practical ML fundamentals track
2. understand metrics, overfitting, and model evaluation
3. build one foundational ML project
4. develop the habit of writing failure analysis, not just success summaries

Deliverables:

- one ML project with README
- one evaluation summary with metrics
- notes explaining what failed and why
- one short note on where this project connects to real user or business value

## Phase 3: Modern Applied AI (Days 91-135)

Goals:

1. learn LLM application basics
2. understand prompting, grounding, and hallucination risks
3. build one LLM or agentic workflow project
4. add trust and evaluation discipline that goes beyond anecdotal testing

Deliverables:

- one AI app project
- prompt or evaluation artifacts
- written analysis of trade-offs and limitations
- a simple trust checklist covering safety, privacy, and fallback behavior

## Phase 4: Production And Career Positioning (Days 136-180)

Goals:

1. harden one project for public presentation
2. create a strong GitHub and case-study presence
3. update resume, profile, and interview stories
4. decide whether the project is best used as a hiring asset, consulting wedge, or product direction

Deliverables:

- 2-3 polished portfolio artifacts
- updated resume and profile summary
- interview-ready stories covering problem, design, evaluation, and lessons learned
- one opportunity thesis or service framing document

## Review Cadence

Every two weeks, answer:

1. What did I build?
2. What did I understand better?
3. What gap still blocks my target role?
4. What proof of progress is visible to another person?
5. Is my work getting closer to real market value, or am I only accumulating knowledge?

# First AI Project Checklist

Use this checklist before calling your first project portfolio-ready.

## Problem Quality

- The problem is real and understandable.
- There is a reason AI is useful here.
- The project is scoped small enough to finish.

## Technical Quality

- The code runs.
- Dependencies are documented.
- Inputs and outputs are clear.
- The README explains setup and usage.

## AI Quality

- The approach is explained.
- The evaluation method is explained.
- Failure cases are acknowledged.
- Limitations are written honestly.

## Portfolio Quality

- The project shows a useful skill for a target AI role.
- The GitHub repo is understandable to another person.
- The story is stronger than “I followed a tutorial.”

## Interview Quality

- I can explain what I built without reading notes.
- I can explain why I chose this design.
- I can explain what failed and how I would improve it.

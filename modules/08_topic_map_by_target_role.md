# Module 8: Topic Map By Target Role

## Why This Module Exists

Different target roles need different topic depth. Learners often waste time by studying everything at the same depth.

Use this map to decide what to go deep on and what to learn at a working level.

## AI Engineer

Go deep on:

1. Python and software engineering
2. prompt engineering and evaluation
3. RAG, tools, agents, and workflow design
4. deployment and product integration

Learn at working depth:

1. classical ML fundamentals
2. neural network and transformer intuition
3. monitoring and observability

## ML Engineer

Go deep on:

1. ML fundamentals and evaluation
2. experimentation and training workflows
3. data pipelines and feature handling
4. deployment and model lifecycle

Learn at working depth:

1. prompting and LLM application patterns
2. product UX trade-offs

## Data Scientist

Go deep on:

1. statistics and experimentation
2. data quality and feature engineering
3. model evaluation and interpretation
4. problem framing and communication

Learn at working depth:

1. software engineering and deployment
2. LLM app design if targeting modern AI roles

## MLOps / AI Platform Engineer

Go deep on:

1. infra and deployment
2. observability and monitoring
3. reproducibility and experiment management
4. CI/CD for models and evals

Learn at working depth:

1. ML and LLM model behavior basics
2. prompting and safety constraints

## AI Product Manager

Go deep on:

1. problem selection and AI use-case framing
2. evaluation criteria and user impact
3. workflow integration and rollout planning
4. responsible AI trade-offs

Learn at working depth:

1. prompting and LLM limitations
2. enough technical depth to challenge implementation choices intelligently

## Practical Use

For your target role, classify each topic as:

- deep
- working knowledge
- awareness only

This prevents unfocused studying.
